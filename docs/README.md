## Documentation Standards

When adding new documentation:

1. Use Markdown format (.md)
2. Include a clear title and table of contents for long documents
3. Use code blocks with language specifiers
4. Include examples and screenshots where applicable
5. Keep documentation up-to-date with code changes
6. Reference related documentation files
